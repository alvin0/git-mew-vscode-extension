# Product Context

## Why This Exists
Developers need fast, contextual Git reviews without leaving VS Code. Git Mew turns diffs into actionable reviews, explanations, diagrams, and descriptions.

## User Experience Goals
- Reviews should feel specific, grounded, and actionable.
- Hidden integration and security risks should be surfaced, not just obvious diff-level comments.
- Output should be structured and easy to scan.

## Key User Flows
- Review Merge: compare two branches and generate a code review or MR description.
- Review Staged Changes: review the current staged diff before committing.
- Review Merged Branch: inspect already-merged work retrospectively.
